    工具安装

    核心架构选择

    目前最主流、性能最强的组合是：

    • 插件管理器: vim-plug (轻量、易用)
    • LSP 客户端: coc.nvim (开箱即用，类似 VS Code 的体验)
    • C++ 语言服务器: clangd (LLVM 官方出品，解析索引最准)
    • 模糊查找: fzf.vim 或 telescope.nvim

    在阅读大型 C++ 工程时，仅仅有 gd (跳转定义) 是不够的，你还需要“跳回来”：

    1. 跳转回原位: 使用 Vim 内置的 Ctrl + o (回到上一个光标位置)，Ctrl + i (去往下一个位置)。这在深层调用链追踪时极其有用。
    2. 分屏阅读: 在跳转定义时，如果想保留当前页面，可以使用 :vsplit 或在 fzf 搜索列表中按 Ctrl + v 打开垂直分屏。
    3. 符号大纲: 输入 :CocList outline 可以看到当前文件的所有类、变量和函数列表，支持实时搜索。

    模式说明

    vim存在三种模式应对不同的编辑需求：普通模式、插入模式、视图模式。打开vim程序默认会进入普通模式，输入i能够切换到插入模式。在插入模式中可以输入任何字符。任何模式中，通过输入ESC返回普通模式。

    普通模式

    字符对应英文的字母，中文的字。以下很多命令可以相互组合。

    • i/a 切换到插入模式，光标位置移到 字符首/字符尾。
    • I/A 切换到插入模式，光标位置移到 行首/行尾。
    • h/l 光标位置 向左/向右 移动1字符。
    • j/k 光标位置 向下/向上 移动1行。
    • w/W 光标移到下一词开始处，大写为词可以包含分隔符
    • e/E 光标移到当前词或下一个词结尾处，大写为词可以包含分隔符
    • b/B 光标移到当前词或下一个词开头处，大写为词可以包含分隔符
    • 0/$ 光标移到 行首/行尾
    • ^/g_ 光标移到当前行 第一个/最后一个 字符
    • f/F<char> 向后/向前 查找字符<char>，查找过程中，通过;/,将光标移到 下/上 一项匹配项。
    • t/T<char> 向后/向前 查找字符<char>，查找过程中，通过;/,将光标移到 下/上 一项匹配项。
    • d<command> 删除光标下的字符，并复制到缓冲区
    • y<command> 复制光标下的字符缓冲区
    • dd 删除当前行的内容，并复制到缓冲区
    • yy 复制当前行的内容到缓冲区
    • x 删除光标下的1个字符，并复制到缓冲区
    • p/P 在当前行 后面/前面 粘贴缓冲区的内容
    • :<command> 执行命令<command>
    • /<string> 搜索字符串<string>，搜索过程中，通过n/N将光标移到 下/上 一项匹配项。
    • % 光标跳转到匹配括号位置处，包括( [ {。若光标不在括号处，则向后查找到第一个括号为匹配对象，然后跳转到匹配括号位置处。
    • * 词搜索。若光标在词<word>内，则等同命令/<word>。否则，向后查找到第一个词为搜索对象。
    • J 合并当前行和下一行，合并时会删除下一行多余的缩进并使用1个空格符作为分隔符。
    • gJ 是J的变体，合并时保留下一行的所有内容。
    • . 重复上一个命令
    • <number><commands> 重复执行<commands>命令列表<number>次
    • u 撤销
    • <Ctrl+r> 重做
    • <number>G 跳转到行<number>
    • gg 等效于命令1G，跳转到第一行
    • G 跳转到最后一行
    • <action>a<object> 或 <action>i<object> 区域操作。<action>支持很多动作，比如：d删除, y复制, v视图模式选择。<object>支持很多目标，比如：w词, s句子, p段落。也支持自然符号，比如：", ', ( , [, {。
    • <</>> 减少/增加 行缩进
    • == 行自动缩进

    命令列表

    • :wq 保存并退出
    • :w 保存
    • :q! 不保存并退出
    • :help <command> 显示关于<command>的帮助信息

    区域操作

    当光标在字符串(map (+) (“foo”))的第一个o字符。

    • vi" → 会选择foo
    • va" → 会选择 "foo".
    • vi) → 会选择 "foo".
    • va) → 会选择("foo")
    • v2i) → 会选择 map (+) ("foo")
    • v2a) → 会选择 (map (+) ("foo"))

    1. 基础分屏创建

    在跳转前，你可以先手动划分空间：

    • :sp (split): 水平分屏。
    • :vsp (vsplit): 垂直分屏（阅读代码时最推荐，左右对比更清晰）。
    • :only: 关闭除当前窗口外的所有窗口，回到全屏阅读。

    2. 联动跳转：在分屏中打开定义

    这是阅读代码的核心痛点：你既想看定义的实现，又不想弄丢现在的上下文。

    配合 fzf.vim 的跳转

    当你使用 :Files 或 :Rg 搜索时：

    • 按下 Enter: 在当前窗口打开。
    • 按下 Ctrl + x: 水平分屏打开。
    • 按下 Ctrl + v: 垂直分屏打开。
    • 按下 Ctrl + t: 新标签页打开。

    3. 跨分屏的高效跳转与导航

    窗口大小调整

    阅读长函数时，可能需要临时拉大窗口：

    • Ctrl+w, =: 所有窗口恢复等大。
    • Ctrl+w, _: 当前窗口高度最大化。
    • Ctrl+w, |: 当前窗口宽度最大化。
    • 10 > 或 10 <: 向左/右微调宽度。

    4. 终极技巧：双栏同步滚动 (Scroll Bind)

    当你需要对比两个相似的 C++ 文件（例如 driver_v1.c 和 driver_v2.c）时，可以使用同步滚动：

    1. 垂直分屏打开两个文件。
    2. 在每个窗口分别输入：:set scrollbind。
    3. 此时滚动其中一个窗口，另一个会同步移动。输入 :set noscrollbind 取消。

    5. 快速预览：不离开主屏幕看定义

    有时候你只是想瞟一眼函数的实现，不想真的跳过去。使用 coc.nvim 的 Float Window (浮窗) 预览：

    • 快捷键: K (对应上条回答中的 doHover 映射)。
    • 进阶: 如果光标在函数名上，执行 :CocAction('showSignatureHelp') 可以直接在浮窗里看参数类型。

    小贴士：习惯使用 Ctrl + o (Jump back) 和 Ctrl + i (Jump forward)。无论你分了多少屏，Vim 都会记录你的光标跳转序列，这是在多文档间反复横跳的“后悔药”。

    3. 分屏与标签页的“联动”

    当你正在分屏阅读一个 C++ 的 .h 和 .cpp 文件，突然想跳到另一个完全不同的模块阅读，但不希望关掉现在的分屏布局时：

    1. 当前窗口转为新标签: 按 Ctrl + w, 接着按 T（大写）。这会将当前焦点的分屏直接“抽离”出来，变成一个新的 Tab。
    2. 在搜索中打开: 如果你使用 fzf.vim 搜索文件，在列表里按 Ctrl + t，选中的文件会直接在新标签页中打开，而不会覆盖你当前的工作区。

    5. 核心逻辑：Buffer、Window 与 Tab 的关系

    理解这三者的关系，能让你在阅读代码时不再迷路：

    • Buffer (缓冲区): 只要你打开过一个文件，它就存在于内存中。即使你关掉了它的窗口，它依然在。
    • Window (窗口): 就是你看到的“分屏”，它是展示 Buffer 的容器。
    • Tab (标签页): 它是 Window 的集合。你可以 Tab 1 看 USB 驱动，Tab 2 看 SDL 渲染代码，互不干扰。

    实用场景提示：在阅读 C++ 代码时，如果你用 gd 跳转到了一个深层函数，看一眼后想保持那个画面并开辟新战场，先执行 :tab split。它会复制当前的布局到一个新标签页，让你放心地继续跳转。

    3. 动态调用栈：调试时的 Stack Trace

    如果你是在运行/调试程序（例如调试 DWC3 控制器逻辑或内核模块），你需要的是真正的运行时堆栈。

    方案 A：使用 Vimspector (全功能 GUI 调试器)

    Vimspector 是 Vim 上最强大的调试插件，支持图形化的堆栈显示。

    1. 安装: Plug 'puremourning/vimspector'
    2. 功能: 触发断点后，它会专门开一个名为 Stack Trace 的窗口，显示当前线程的调用深度，点击即可跳转。

    方案 B：内置 Termdebug (GDB 联动)

    如果你不想装重型插件，Vim 8.1+ 自带了 termdebug：

    1. 在 Vim 中输入 :packadd termdebug。
    2. 输入 :Termdebug [你的可执行程序]。
    3. 在 GDB 窗口运行程序并在断点停止。
    4. 在 GDB 窗口输入 bt (backtrace)，Vim 会实时联动源码窗口。

    4. 终极技巧：使用导图式导航

    对于特别复杂的 C++ 类调用关系，理解 Buffer (缓冲区) -> Window (窗口) -> Tab (标签页) 的逻辑能帮你手动“标记”调用栈：

    • 标记位置: 在跳转前按 ma（做标记 a）。
    • 深层跳转: gd 跳进函数。
    • 回溯: 按 'a（回到标记 a）或者使用内置的 Tag Stack。
